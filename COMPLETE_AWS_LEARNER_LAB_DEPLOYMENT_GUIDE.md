# AWS Learner Lab: Diet-diary Deployment Documentation

This guide provides a step-by-step walkthrough to deploy the modernized **Diet-diary** application to your **AWS Learner Lab** environment. 

> [!IMPORTANT]
> **Lab Constraints Reminder**: 
> - You **must** use the `LabRole` for all permissions.
> - Stick to `us-east-1` (North Virginia) to ensure full service availability.
> - Always use the `AWS_SESSION_TOKEN` since lab credentials expire.

---

## 🏗️ Phase 1: Resource Provisioning (The AWS Console)

### 1. Amazon S3 (Binary Storage)
1. Go to **S3** > **Create bucket**.
2. **Bucket name**: `diet-diary-uploads-lab-[your-id]` (must be unique).
3. **Region**: `us-east-1`.
4. **Settings**: Keep "Block all public access" **ON**. Our app uses signed URLs for security.

### 2. Amazon RDS (Database)
1. Go to **RDS** > **Create database**.
2. **Engine**: MySQL (8.0.x).
3. **Templates**: **Free Tier**.
4. **Identifier**: `diet-diary-db`.
5. **Credentials**: 
   - Master username: `admin`
   - Master password: `YourSecurePassword123`
6. **Public access**: **Yes** (this allows you to test from your local machine).
7. **Security Group**: Ensure it allows inbound traffic on port **3306** from your IP.

### 3. Amazon SNS (Alerts)
1. Go to **SNS** > **Topics** > **Create topic**.
2. **Type**: Standard.
3. **Name**: `DietDiaryNotifications`.
4. Once created, go to **Subscriptions** > **Create subscription**.
   - Protocol: **Email**.
   - Endpoint: Your email address.
5. **CRITICAL**: Open your email and click **Confirm Subscription**.

### 4. AWS Secrets Manager (Security)
1. Go to **Secrets Manager** > **Store a new secret**.
2. **Secret type**: "Other type of secret".
3. **Key/Value pairs**:
   | Key | Value |
   | :--- | :--- |
   | `DB_HOST` | [Your RDS Endpoint] |
   | `DB_USER` | `admin` |
   | `DB_PASSWORD` | `YourSecurePassword123` |
   | `DB_NAME` | `diet_diary` |
   | `S3_BUCKET` | [Your S3 Bucket Name] |
   | `SNS_TOPIC_ARN` | [Your SNS Topic ARN] |
4. **Secret name**: `DietDiary/Prod`.

## 💻 Phase 2: Application Configuration

### 1. Local Environment Setup
Create a `.env` file in the project root:
```env
PORT=3000
AWS_REGION=us-east-1
AWS_SECRET_NAME=DietDiary/Prod
NODE_ENV=production

# For Local Testing (Copy these from your Lab Console "AWS Details" tab)
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AWS_SESSION_TOKEN=...
```

### 2. Install Project Dependencies
Run this in your terminal:
```bash
npm install
```

## 🚀 Phase 3: Hosting on Amazon EC2

1. **Launch Instance**: 
   - Name: `DietDiaryServer`.
   - AMI: **Amazon Linux 2023**.
   - IAM Instance Profile: **`LabInstanceProfile`** (Required for the app to talk to S3/Secrets).
2. **Security Group**: Allow Port **80** (HTTP) and **3000** (App).
3. **Install Dependencies**:
   ```bash
   sudo yum update -y
   sudo yum install -y nodejs git
   ```
4. **Clone & Run**:
   ```bash
   git clone [your-repo-url]
   cd aws
   npm install
   npm start
   ```

## ❓ Troubleshooting (Self-Test Check)

> [!CAUTION]
> **Avoid the "File Not Found" Error**: 
> Do **NOT** open `index.html` directly in your browser using "Right Click > Open With". This will break the API calls.
> 
> **Correct Way**:
> 1. In your terminal, run `npm start`.
> 2. Open your browser and go to `http://localhost:3000`.

### Common Health Checks
- **RDS Connectivity**: Use `node test-all-services.js` to verify DB connection.
- **S3 Uploads**: Check the "Network" tab in Chrome DevTools to see if binary data is flowing to Amazon.
